## 二、核心功能指令（STM32 发送帧直接抄）

### 1. 修改设备地址（功能码 0x06 写单个寄存器）

#### 指令说明

- 目标寄存器：`0x07D0`（对应十进制 42001，设备地址寄存器）

- 指令格式（以「地址 1 改地址 2」为例）：

|原地址|功能码|起始寄存器高|起始寄存器低|新地址高|新地址低|CRC16 低|CRC16 高|
|---|---|---|---|---|---|---|---|
|0x01|0x06|0x07|0xD0|0x00|0x02|0x08|0x86|
- 完整发送帧：`01 06 07 D0 00 02 08 86`

- 响应规则：传感器接收正确后，**原样返回该帧**

- 广播地址用法：忘记原地址时，用 `0xFF` 代替原地址，此时主机只能接 1 个从机，可用于地址查询

#### 通用改地址帧模板

```Plain Text

[原地址] 06 07 D0 00 [新地址] [CRC16低] [CRC16高]
```

## 四、数据计算规则（STM32 代码核心转换逻辑）

### 1. 温度计算（重点：负数补码处理）

- 正数：直接转十进制后 ÷10

- 负数：以补码形式传输，需先转原码再计算

    - 示例：`0xFF9B`（十六进制）

        - 补码转十进制：`0xFF9B = -101`

        - 实际温度：`-101 / 10 = -10.1℃`

- 代码逻辑：

    ```C
    
    int16_t temp_raw = (data[3] << 8) | data[4]; // 读取2字节温度
    float temp = temp_raw / 10.0f; // 直接计算，int16_t自动处理补码
    ```

### 2. 含水率计算

- 示例：`0x0292`（十六进制）= `658`（十进制）

- 实际含水率：`658 / 10 = 65.8%`

### 3. 电导率计算

- 示例：`0x03E8`（十六进制）= `1000`（十进制）

- 实际电导率：`1000 us/cm`

### 4. PH值计算

- 示例：`0x0038`（十六进制）= `56`（十进制）

- 实际PH值：`56 / 10 = 5.6`

## 五、STM32 开发关键注意事项

### 1. 硬件层

- RS485 收发控制：STM32 需用 GPIO 控制 DE/RE 引脚，**发送时拉高，接收时拉低**

- 串口波特率：默认 4800，修改波特率后需同步更新 STM32 UART 配置

- 总线隔离：长距离通信建议加隔离芯片，避免干扰

### 2. 协议层

- CRC16 校验：必须实现 Modbus 标准 CRC16 算法，所有发送/接收帧都要校验

- 帧超时处理：Modbus-RTU 要求帧间间隔 ≥ 3.5 个字符时间，STM32 串口需做超时判断

- 地址冲突：多传感器组网时，必须先修改地址再接入总线，避免地址冲突

### 3. 代码层

- 状态机设计：用状态机处理「发送指令→等待响应→解析数据→校验CRC」流程

- 数据缓存：用环形缓冲区缓存串口数据，避免丢帧

- 异常处理：超时、CRC 错误、功能码不匹配等情况，做重发机制

```C
uint8_t modbus_read_cmd[8] = {0x01, 0x03, 0x00, 0x00, 0x00, 0x04, 0x44, 0x09};

```

## 七、常见问题排查

1. **无响应**：检查 RS485 收发引脚电平、波特率/校验位配置、CRC 校验是否正确

2. **温度负数异常**：确认用 `int16_t` 接收温度数据，不要用 `uint16_t`

3. **地址修改失败**：确认原地址正确，广播地址 0xFF 仅支持单从机

4. **数据乱码**：检查串口波特率、帧间隔是否符合 Modbus-RTU 要求
