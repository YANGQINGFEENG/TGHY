# 数据库结构文档

## 1. 数据库概述

本系统使用MySQL数据库存储设备信息和传感器数据，数据库名为`agriculture_iot`。

## 2. 表结构设计

### 2.1 设备表（devices）

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| id | INT | AUTO_INCREMENT PRIMARY KEY | 设备ID |
| device_id | VARCHAR(50) | NOT NULL UNIQUE | 设备唯一标识 |
| device_name | VARCHAR(100) | NOT NULL | 设备名称 |
| device_type | ENUM('environment', 'soil', 'water', 'security') | NOT NULL | 设备类型 |
| ip_address | VARCHAR(50) | | 设备IP地址 |
| last_seen | DATETIME | | 设备最后在线时间 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 设备创建时间 |

**说明**：
- `device_id`：设备的唯一标识符，由ESP32设备提供
- `device_type`：设备类型，分为环境、土壤、水质和安全四类
- `last_seen`：用于判断设备是否在线，由系统自动更新
- `created_at`：设备首次注册时间

### 2.2 传感器数据表（sensor_data）

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| id | INT | AUTO_INCREMENT PRIMARY KEY | 数据ID |
| device_id | VARCHAR(50) | NOT NULL | 设备ID |
| timestamp | DATETIME | NOT NULL | 数据采集时间 |
| sensor_type | VARCHAR(50) | NOT NULL | 传感器类型 |
| value | DECIMAL(10,2) | NOT NULL | 传感器值 |
| unit | VARCHAR(20) | NOT NULL | 单位 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 数据存储时间 |

**说明**：
- `device_id`：关联到devices表的device_id字段
- `timestamp`：传感器数据采集的时间
- `sensor_type`：传感器类型，如temperature、humidity等
- `value`：传感器采集的数值
- `unit`：数值的单位，如℃、%、ppm等

## 3. 索引设计

### 3.1 设备表索引
- 主键索引：`id`
- 唯一索引：`device_id`

### 3.2 传感器数据表索引
- 主键索引：`id`
- 普通索引：`device_id` - 加速设备数据查询
- 普通索引：`timestamp` - 加速时间范围查询
- 普通索引：`sensor_type` - 加速传感器类型查询

## 4. 数据关系

- **devices表**与**sensor_data表**是一对多关系
- 一个设备可以产生多条传感器数据
- 每条传感器数据都属于一个设备

## 5. 数据流程

1. ESP32设备通过TCP协议发送数据到服务器
2. 服务器解析数据，提取设备信息和传感器数据
3. 检查设备是否已注册，若未注册则插入新设备记录
4. 更新设备的最后在线时间和IP地址
5. 插入传感器数据记录
6. 前端通过API查询设备和传感器数据

## 9. 数据库初始化

使用以下SQL语句初始化数据库：

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS agriculture_iot CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 使用数据库
USE agriculture_iot;

-- 创建设备表
CREATE TABLE IF NOT EXISTS devices (
    id INT AUTO_INCREMENT PRIMARY KEY,
    device_id VARCHAR(50) NOT NULL UNIQUE,
    device_name VARCHAR(100) NOT NULL,
    device_type ENUM('environment', 'soil', 'water', 'security') NOT NULL,
    ip_address VARCHAR(50),
    last_seen DATETIME,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 创建传感器数据表
CREATE TABLE IF NOT EXISTS sensor_data (
    id INT AUTO_INCREMENT PRIMARY KEY,
    device_id VARCHAR(50) NOT NULL,
    timestamp DATETIME NOT NULL,
    sensor_type VARCHAR(50) NOT NULL,
    value DECIMAL(10,2) NOT NULL,
    unit VARCHAR(20) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (device_id) REFERENCES devices(device_id)
);

-- 创建索引
CREATE INDEX idx_sensor_data_device_id ON sensor_data(device_id);
CREATE INDEX idx_sensor_data_timestamp ON sensor_data(timestamp);
CREATE INDEX idx_sensor_data_sensor_type ON sensor_data(sensor_type);
```
