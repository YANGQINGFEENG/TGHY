# 系统启动流程文档

## 1. 系统启动概述

本系统的启动流程包括数据库初始化、后端服务启动和前端页面访问三个主要步骤。本文档详细说明每个步骤的操作流程和注意事项。

## 2. 启动前准备

### 2.1 环境要求

- **操作系统**：Windows 10/11
- **Python**：3.7+
- **MySQL**：5.7+
- **网络**：局域网环境

### 2.2 软件安装

1. **安装Python**
   - 从官网下载并安装Python 3.7或更高版本
   - 确保Python已添加到系统环境变量

2. **安装MySQL**
   - 下载并安装MySQL Community Server
   - 记住MySQL的root用户密码

3. **安装依赖库**
   ```bash
   cd backend
   venv\Scripts\activate
   pip install -r requirements.txt
   ```

## 3. 数据库初始化

### 3.1 创建数据库

1. **登录MySQL**
   ```bash
   mysql -u root -p
   ```

2. **创建数据库**
   ```sql
   CREATE DATABASE IF NOT EXISTS agriculture_iot CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

3. **退出MySQL**
   ```sql
   exit;
   ```

### 3.2 导入表结构

```bash
mysql -u root -p agriculture_iot < backend\database\schema.sql
```

## 4. 后端服务启动

### 4.1 配置系统

1. **修改配置文件**
   编辑 `backend\config.py` 文件，根据实际情况修改以下配置：

   ```python
   # 数据库配置
   DATABASE_CONFIG = {
       'host': 'localhost',
       'port': 3306,
       'user': 'root',
       'password': 'your_password',  # 修改为你的MySQL密码
       'database': 'agriculture_iot'
   }

   # TCP服务器配置
   TCP_SERVER_CONFIG = {
       'host': '0.0.0.0',
       'port': 8888
   }

   # API服务器配置
   API_SERVER_CONFIG = {
       'host': '0.0.0.0',
       'port': 5000,
       'debug': False
   }
   ```

### 4.2 启动服务

1. **手动启动**
   ```bash
   cd backend
   venv\Scripts\activate
   python main.py
   ```

2. **启动成功标志**
   当看到以下输出时，表示服务启动成功：
   ```
   INFO:tcp_server:TCP服务器启动成功，监听地址: 0.0.0.0:8888
   * Serving Flask app 'api.api'
   * Debug mode: off
   ```

### 4.3 验证服务

1. **检查TCP服务器**
   - 端口8888是否已监听
   - 可以使用telnet测试连接

2. **检查API服务**
   - 访问 `http://localhost:5000/api/devices`
   - 应返回JSON格式的设备列表

## 5. 前端页面访问

1. **直接打开前端页面**
   - 双击 `frontend\index.html` 文件
   - 或在浏览器中输入文件路径

2. **页面功能**
   - 设备状态概览：显示所有设备的在线状态
   - 数据可视化：通过图表展示传感器数据趋势
   - 实时数据：显示最新的传感器数据

## 6. 自启动配置

### 6.1 创建启动脚本

1. **创建批处理文件**
   在 `backend` 目录下创建 `start_server.bat` 文件：

   ```batch
   @echo off
   cd /d %~dp0
   venv\Scripts\activate
   python main.py
   ```

2. **创建快捷方式**
   - 右键点击 `start_server.bat` 文件
   - 选择 "创建快捷方式"

### 6.2 设置Windows自启动

1. **方法一：启动文件夹**
   - 按下 `Win + R` 组合键
   - 输入 `shell:startup` 并回车
   - 将快捷方式复制到启动文件夹

2. **方法二：任务计划程序**
   - 打开 "任务计划程序"
   - 点击 "创建基本任务"
   - 按照向导设置任务名称和触发器
   - 操作选择 "启动程序"
   - 浏览选择 `start_server.bat` 文件
   - 完成设置

### 6.3 自启动验证

1. **重启计算机**
   - 重启计算机后，服务应自动启动

2. **检查服务状态**
   - 检查端口8888和5000是否已监听
   - 访问前端页面确认数据是否正常显示

## 7. 服务管理

### 7.1 停止服务

1. **手动停止**
   - 在命令窗口中按 `Ctrl + C`

2. **任务管理器停止**
   - 打开任务管理器
   - 找到Python进程并结束

### 7.2 重启服务

1. **手动重启**
   ```bash
   cd backend
   venv\Scripts\activate
   python main.py
   ```

2. **自启动重启**
   - 重启计算机或重启任务计划程序中的任务

## 8. 故障排查

### 8.1 常见启动问题

1. **数据库连接失败**
   - 检查MySQL服务是否运行
   - 检查数据库配置是否正确
   - 检查数据库用户权限

2. **端口被占用**
   - 检查端口8888和5000是否被其他程序占用
   - 修改配置文件中的端口号

3. **依赖库缺失**
   - 重新安装依赖库：`pip install -r requirements.txt`

4. **文件路径错误**
   - 确保所有文件路径正确
   - 检查日志文件路径是否存在

### 8.2 日志查看

系统日志存储在 `logs` 目录下：
- `main.log`：主程序日志
- `tcp_server.log`：TCP服务器日志
- `database.log`：数据库操作日志
- `data_handler.log`：数据处理日志
- `api.log`：API接口日志

通过查看日志文件，可以了解系统运行状态和错误信息。

## 9. 性能优化

### 9.1 启动速度优化

   - 减少不必要的初始化操作

2. **内存使用优化**

   - 优化数据缓存策略

### 9.2 系统稳定性

1. **自动重连**
   - 实现数据库自动重连
   - 实现TCP服务器自动重连

2. **错误处理**
   - 完善错误处理机制
   - 增加系统容错能力

## 10. 部署建议

### 10.1 生产环境部署

1. **服务器配置**
   - 选择性能合适的服务器
   - 确保网络稳定

2. **安全配置**
   - 配置防火墙
   - 实现访问控制
   - 使用HTTPS加密传输

3. **监控系统**
   - 监控服务运行状态
   - 设置告警机制

### 10.2 多设备部署

1. **网络规划**
   - 确保ESP32设备能够连接到服务器
   - 考虑网络带宽和延迟

2. **负载均衡**
   - 当设备数量较多时，考虑使用负载均衡
   - 优化数据库查询性能

## 11. 总结

本系统的启动流程包括以下步骤：

1. **环境准备**：安装Python、MySQL和依赖库
2. **数据库初始化**：创建数据库和表结构
3. **服务配置**：修改配置文件中的参数
4. **服务启动**：启动后端服务
5. **前端访问**：打开前端页面
6. **自启动配置**：设置系统自启动

通过本文档的指导，您可以快速部署和管理农业物联网监控系统，确保系统稳定运行。
