# 农业物联网监控系统文档

## 1. 项目概述

本项目是一个完整的农业物联网监控系统，包括前端、后端和硬件三部分。系统通过ESP32设备采集农业环境数据，通过TCP协议传输到服务器，服务器将数据存储到MySQL数据库中，并通过Web前端展示给用户。

### 1.1 系统架构

- **硬件层**：ESP32设备，通过WiFi连接到局域网，采集环境数据
- **传输层**：TCP/IP协议，负责数据传输
- **服务层**：Python后端服务，包括TCP服务器、数据处理和API服务
- **存储层**：MySQL数据库，存储设备信息和传感器数据
- **展示层**：Web前端，展示设备状态和数据可视化

### 1.2 系统功能

- 实时接收ESP32设备发送的数据
- 存储设备信息和传感器数据到数据库
- 提供API接口查询设备和数据
- 展示设备状态概览
- 数据可视化图表
- 实时数据表格

## 4. 安装与部署

### 4.1 后端服务

1. **创建数据库**
   - 在MySQL中创建名为`agriculture_iot`的数据库
   - 运行`backend/database/schema.sql`创建表结构

2. **安装依赖**
   ```bash
   cd backend
   venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. **配置系统**
   - 修改`backend/config.py`中的配置参数

4. **启动服务**
   ```bash
   venv\Scripts\activate
   python main.py
   ```

### 4.2 前端页面

- 直接打开`frontend/index.html`文件
- 确保后端服务正在运行

### 4.3 ESP32设备

- 配置ESP32连接到同一局域网
- 编写ESP32代码，通过TCP协议发送数据到服务器
- 数据格式示例：
  ```json
  {
    "device_id": "esp32_001",
    "device_name": "环境传感器",
    "device_type": "environment",
    "timestamp": "2026-04-02T12:00:00",
    "sensors": {
      "temperature": {"value": 25.5, "unit": "℃"},
      "humidity": {"value": 60.0, "unit": "%"}
    }
  }
  ```

## 5. API接口

### 5.1 获取设备列表

- **URL**: `http://localhost:5000/api/devices`
- **方法**: GET
- **返回**: 设备列表

### 5.2 获取设备详情

- **URL**: `http://localhost:5000/api/devices/{device_id}`
- **方法**: GET
- **返回**: 设备详细信息

### 5.3 获取传感器数据

- **URL**: `http://localhost:5000/api/sensor-data`
- **方法**: GET
- **参数**:
  - `device_id`: 设备ID（可选）
  - `sensor_type`: 传感器类型（可选）
  - `limit`: 返回数据条数（默认100）
- **返回**: 传感器数据列表

### 5.4 获取最新传感器数据

- **URL**: `http://localhost:5000/api/sensor-data/latest`
- **方法**: GET
- **参数**:
  - `device_id`: 设备ID（可选）
  - `sensor_type`: 传感器类型（可选）
- **返回**: 最新的传感器数据

## 8. 故障排查

### 8.1 常见问题

1. **TCP连接失败**
   - 检查ESP32是否连接到正确的WiFi网络
   - 检查服务器IP地址和端口是否正确
   - 检查防火墙是否允许端口访问

2. **数据库连接失败**
   - 检查MySQL服务是否运行
   - 检查数据库配置是否正确
   - 检查数据库用户权限

3. **API接口无法访问**
   - 检查后端服务是否运行
   - 检查API服务器配置
   - 检查防火墙设置

### 8.2 日志查看

通过查看相应的日志文件，可以了解系统运行状态和错误信息。
