# 硬件数据注入方案

## 1. 项目概述

本方案旨在实现STM32硬件设备通过WiFi模块向智能农业服务器注入传感器数据。系统已成功连接到本地WiFi网络，现在需要实现数据的实时上传功能。

## 2. 服务器API接口分析

### 2.1 现有API接口

| 接口路径 | 方法 | 功能 | 请求体格式 | 响应格式 |
|---------|------|------|------------|----------|
| `/api/sensors` | GET | 获取所有传感器 | N/A | `{"success": true, "data": [...], "total": 10}` |
| `/api/sensors` | POST | 新增传感器 | `{"name": "传感器名称", "type_id": 1, "location": "位置"}` | `{"success": true, "data": {...}}` |
| `/api/sensors/[id]/data` | GET | 获取传感器数据 | N/A | `{"success": true, "data": [...], "stats": {...}, "total": 100}` |
| `/api/sensors/[id]/data` | POST | 新增传感器数据 | `{"value": 25.5}` | `{"success": true, "data": {...}}` |

### 2.2 传感器类型

| 类型ID | 类型 | 名称 | 单位 |
|--------|------|------|------|
| 1 | temperature | 温度传感器 | °C |
| 2 | humidity | 空气湿度传感器 | % |
| 3 | light | 光照传感器 | Lux |
| 4 | soil | 土壤湿度传感器 | % |

## 3. 硬件数据注入方案

### 3.1 网络连接

1. **WiFi连接**：
   - 已成功连接到本地WiFi网络
   - SSID: `8848`
   - 密码: `12345678`

2. **服务器连接**：
   - 服务器地址: 本地服务器IP地址（如 `192.168.1.100`）
   - 端口: `3000`（Next.js默认端口）
   - 协议: HTTP

### 3.2 数据发送策略

1. **发送频率**：
   - 常规数据：每2分钟发送一次
   - 异常数据：当传感器数据超出阈值时立即发送

2. **数据格式**：
   - 使用HTTP POST请求
   - Content-Type: application/json
   - 每个传感器单独发送数据

3. **传感器ID映射**：

| 硬件传感器 | 服务器传感器ID | 类型 |
|-----------|----------------|------|
| DHT11温度 | T-001 | temperature |
| DHT11湿度 | H-001 | humidity |
| 光照传感器 | L-001 | light |
| 土壤湿度 | S-001 | soil |
| 土壤温度 | T-002 | temperature |
| 土壤电导率 | 需新增 | conductivity |
| 土壤PH值 | 需新增 | ph |

### 3.3 HTTP请求示例

#### 发送温度数据
```http
POST http://192.168.1.100:3000/api/sensors/T-001/data
Content-Type: application/json

{
  "value": 25.5
}
```

#### 发送湿度数据
```http
POST http://192.168.1.100:3000/api/sensors/H-001/data
Content-Type: application/json

{
  "value": 60.0
}
```

## 4. 硬件实现方案

### 4.1 代码结构

1. **新增文件**：
   - `Hardware/ServerComm.c` - 服务器通信模块
   - `Hardware/ServerComm.h` - 服务器通信头文件

2. **修改文件**：
   - `User/main.c` - 添加数据发送逻辑

### 4.2 核心功能实现

1. **服务器通信模块**：
   - `ServerComm_Init()` - 初始化服务器通信
   - `ServerComm_SendSensorData()` - 发送传感器数据
   - `ServerComm_CheckConnection()` - 检查服务器连接状态

2. **数据发送逻辑**：
   - 读取传感器数据
   - 构建JSON数据
   - 发送HTTP POST请求
   - 处理服务器响应
   - 错误重试机制

### 4.3 错误处理

1. **网络错误**：
   - WiFi连接失败：重新连接
   - 服务器连接失败：重试3次
   - HTTP请求失败：记录错误并重试

2. **数据错误**：
   - 传感器数据异常：标记为无效数据
   - JSON构建失败：使用默认值
