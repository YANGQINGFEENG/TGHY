# 网页端控制硬件执行器功能

## 功能概述

实现了网页端远程控制硬件执行器的功能。用户可以在网页上点击执行器的开关按钮，控制指令会通过服务器传递给硬件端执行。

---

## 架构设计

### 数据流程

```
网页端 → 服务器数据库 → 硬件端 → 执行设备
  ↓           ↓            ↓          ↓
发送指令   存储指令     查询指令    执行指令
  ↓           ↓            ↓          ↓
更新状态   确认完成     确认结果    反馈状态
```

### 技术方案

采用**轮询方式**：
1. 网页端发送控制指令到服务器
2. 服务器将指令存入数据库（状态：pending）
3. 硬件端定期查询服务器获取待执行指令
4. 硬件端执行指令并更新数据库（状态：executed）
5. 服务器更新执行器状态

---

## 数据库设计

### 控制指令表（actuator_commands）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 指令ID（主键） |
| actuator_id | VARCHAR(20) | 执行器ID |
| command | ENUM('on', 'off') | 控制命令 |
| status | ENUM('pending', 'executed', 'failed') | 指令状态 |
| created_at | TIMESTAMP | 创建时间 |
| executed_at | TIMESTAMP | 执行时间 |

## API接口

### 1. 发送控制指令（网页端）

**POST /api/actuators/[id]/commands**

请求示例：
```json
{
  "command": "on"
}
```

响应示例：
```json
{
  "success": true,
  "data": {
    "id": 1,
    "actuator_id": "WP-002",
    "command": "on",
    "status": "pending"
  },
  "message": "OK"
}
```

### 2. 查询控制指令（硬件端）

**GET /api/actuators/[id]/commands**

响应示例：
```json
{
  "success": true,
  "data": {
    "id": 1,
    "actuator_id": "WP-002",
    "command": "on",
    "status": "pending",
    "created_at": "2026-04-13T04:00:00.000Z"
  },
  "message": "OK"
}
```

### 3. 确认指令执行（硬件端）

**PATCH /api/actuators/[id]/commands**

请求示例：
```json
{
  "command_id": 1,
  "status": "executed"
}
```

响应示例：
```json
{
  "success": true,
  "message": "OK"
}
```

## 硬件端集成

### 1. 添加文件

将 `ServerComm_Commands.c` 添加到项目中。

### 2. 在主循环中调用

```c
// 在主循环中定期查询控制指令
if (g_wifi_connected && ServerComm_IsConnected()) {
    ServerComm_CheckAndExecuteCommand(ACTUATOR_ID_PUMP);
    delay_ms(1000); // 每秒查询一次
}
```

### 3. 函数说明

```c
// 查询并执行控制指令
uint8_t ServerComm_CheckAndExecuteCommand(const char *actuator_id);

// 确认指令执行结果
uint8_t ServerComm_ConfirmCommand(const char *actuator_id, int command_id, const char *status);
```

## 前端使用

### 1. 打开网页

访问 http://localhost:3000 或 http://26.162.225.180:3000

### 2. 找到执行器控制区域

在主页找到"执行器状态"卡片

### 3. 点击开关按钮

点击执行器的开关按钮，系统会：
1. 发送控制指令到服务器
2. 更新数据库状态
3. 等待硬件端查询并执行
4. 自动刷新执行器状态

## 注意事项

1. **硬件端查询频率**：建议每1-2秒查询一次，避免过于频繁
2. **指令超时**：如果指令长时间未执行，可以手动清理数据库中的pending指令
3. **网络稳定性**：确保硬件端和服务器之间的网络连接稳定
4. **并发控制**：同一时间只能有一个待执行的指令

## 故障排查

### 问题1：网页端发送指令失败

检查：
- 服务器是否运行
- 执行器ID是否正确
- 网络连接是否正常

### 问题2：硬件端查询不到指令

检查：
- 硬件端网络连接是否正常
- 执行器ID是否匹配
- 数据库中是否有pending状态的指令

### 问题3：指令执行后状态未更新

检查：
- 硬件端是否正确调用确认接口
- 数据库更新是否成功
- 前端是否自动刷新
