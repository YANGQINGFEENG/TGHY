# 智慧农业物联网监控平台项目说明文档

## 1. 项目概述

智慧农业物联网监控平台是一个基于Next.js的现代化Web应用，用于实时监控和管理农业环境传感器数据。该平台提供了直观的仪表盘界面，展示温度、湿度、光照和土壤湿度等关键农业环境参数，并支持数据趋势分析、设备管理和数据导出等功能。

### 主要功能
- 实时数据监控与展示
- 传感器数据趋势分析
- 设备状态管理
- 数据导出功能
- 多类型传感器支持
- 响应式设计，适配不同设备

## 2. 技术栈

### 前端技术
- **框架**: Next.js 16.2.0
- **语言**: TypeScript
- **UI组件**: shadcn/ui
- **样式**: Tailwind CSS v4.2.0
- **图表库**: Recharts
- **图标库**: Lucide React
- **状态管理**: React useState/useEffect

### 后端技术
- **API框架**: Next.js API Routes
- **数据库**: MySQL (设计)
- **数据处理**: 模拟数据 (实际部署时使用真实数据库)

## 4. 数据库设计

### 4.1 表结构

#### sensor_types表
| 字段名 | 数据类型 | 约束 | 描述 |
|--------|---------|------|------|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 传感器类型ID |
| type | VARCHAR(50) | NOT NULL UNIQUE | 传感器类型标识 |
| name | VARCHAR(100) | NOT NULL | 传感器类型名称 |
| unit | VARCHAR(20) | NOT NULL | 传感器单位 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

#### sensors表
| 字段名 | 数据类型 | 约束 | 描述 |
|--------|---------|------|------|
| id | VARCHAR(20) | PRIMARY KEY | 传感器ID |
| name | VARCHAR(100) | NOT NULL | 传感器名称 |
| type_id | INT | NOT NULL, FOREIGN KEY | 传感器类型ID |
| location | VARCHAR(255) | NOT NULL | 传感器位置 |
| status | ENUM('online', 'offline') | DEFAULT 'offline' | 传感器状态 |
| battery | INT | DEFAULT 100 | 电池电量 |
| last_update | TIMESTAMP | NULL | 最后更新时间 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

#### sensor_data表
| 字段名 | 数据类型 | 约束 | 描述 |
|--------|---------|------|------|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 数据ID |
| sensor_id | VARCHAR(20) | NOT NULL, FOREIGN KEY | 传感器ID |
| value | DECIMAL(10, 2) | NOT NULL | 传感器数值 |
| timestamp | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 数据采集时间 |

#### device_status_history表
| 字段名 | 数据类型 | 约束 | 描述 |
|--------|---------|------|------|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 历史记录ID |
| sensor_id | VARCHAR(20) | NOT NULL, FOREIGN KEY | 传感器ID |
| status | ENUM('online', 'offline') | NOT NULL | 状态 |
| timestamp | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 状态变更时间 |

### 4.2 初始数据

- **传感器类型**: 温度、湿度、光照、土壤湿度
- **传感器设备**: 10个示例传感器，分布在不同区域
- **模拟数据**: 每个传感器24小时的历史数据

## 5. API接口文档

### 5.1 传感器类型接口

#### GET /api/sensor-types
- **功能**: 获取所有传感器类型
- **参数**: 无
- **返回格式**:
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "type": "temperature",
      "name": "温度传感器",
      "unit": "°C"
    }
  ],
  "total": 4
}
```

#### POST /api/sensor-types
- **功能**: 新增传感器类型
- **参数**:
  - type: 传感器类型标识
  - name: 传感器类型名称
  - unit: 传感器单位
- **返回格式**:
```json
{
  "success": true,
  "data": {
    "id": 5,
    "type": "new_type",
    "name": "新传感器类型",
    "unit": "单位"
  }
}
```

### 5.2 传感器接口

#### GET /api/sensors
- **功能**: 获取所有传感器
- **参数**:
  - type: 可选，按传感器类型过滤
- **返回格式**:
```json
{
  "success": true,
  "data": [
    {
      "id": "T-001",
      "name": "A区温室1号温度传感器",
      "type_id": 1,
      "location": "A区温室",
      "status": "online",
      "battery": 95,
      "last_update": "2026-04-03T15:00:00Z",
      "type": "temperature",
      "type_name": "温度传感器",
      "unit": "°C"
    }
  ],
  "total": 10
}
```

#### POST /api/sensors
- **功能**: 新增传感器
- **参数**:
  - name: 传感器名称
  - type_id: 传感器类型ID
  - location: 传感器位置
- **返回格式**:
```json
{
  "success": true,
  "data": {
    "id": "T-004",
    "name": "新温度传感器",
    "type_id": 1,
    "location": "D区大棚",
    "status": "offline",
    "battery": 100,
    "last_update": null
  }
}
```

### 5.3 传感器数据接口

#### GET /api/sensors/[id]/data
- **功能**: 获取传感器数据
- **参数**:
  - startTime: 可选，开始时间
  - endTime: 可选，结束时间
  - limit: 可选，返回数据条数限制
- **返回格式**:
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "sensor_id": "T-001",
      "value": 24.5,
      "timestamp": "2026-04-03T15:00:00Z"
    }
  ],
  "stats": {
    "avg": 24.2,
    "max": 28.5,
    "min": 18.2
  },
  "total": 24
}
```

#### POST /api/sensors/[id]/data
- **功能**: 新增传感器数据
- **参数**:
  - value: 传感器数值
- **返回格式**:
```json
{
  "success": true,
  "data": {
    "id": 241,
    "sensor_id": "T-001",
    "value": 25.3,
    "timestamp": "2026-04-03T16:00:00Z"
  }
}
```

## 6. 部署指南

### 6.1 环境要求
- Node.js 18+
- MySQL 8.0+
- npm 或 yarn

### 6.2 安装步骤
1. **克隆项目**:
   ```bash
   git clone <项目仓库地址>
   cd <项目目录>
   ```

2. **安装依赖**:
   ```bash
   npm install
   ```

3. **配置数据库**:
   - 启动MySQL服务
   - 执行数据库脚本:
     ```bash
     mysql -u root -p < database.sql
     ```

4. **配置环境变量**:
   - 创建 `.env` 文件，添加数据库连接信息:
     ```
     DATABASE_URL="mysql://root:password@localhost:3306/smart_agriculture"
     ```

5. **启动开发服务器**:
   ```bash
   npm run dev
   ```

6. **构建生产版本**:
   ```bash
   npm run build
   npm start
   ```
