# 临时测试文件管理 Skill

## 触发词
- "管理临时测试文件"
- "清理临时文件"
- "创建临时测试文件"
- "临时文件归档"
- "临时文件保留期限"

## 输入
- 临时文件类型（测试脚本/测试数据/调试输出/临时备份/其他）
- 测试阶段（单元测试/集成测试/系统测试/验收测试/里程碑测试）
- 文件用途描述
- 预期使用期限
- 特殊保留需求（可选）

## 输出
- 临时文件创建结果
- 清理建议
- 归档状态
- 保留期限提醒
- 违规警告

## 执行步骤

### 1. 临时文件创建
1. **文件命名**：按照规范生成文件名
   ```
   [前缀]_[创建日期]_[描述]_[创建者]_[版本/序号].[扩展名]
   ```

2. **文件头注释**：添加临时文件声明
   ```python
   # ============================================================
   # 临时测试文件声明
   # ============================================================
   # 文件用途: [简要描述文件用途]
   # 创建日期: [YYYY-MM-DD]
   # 创建者:   [创建者姓名]
   # 预期到期: [YYYY-MM-DD]
   # 关联任务: [关联的任务或 Issue ID，可选]
   # 自动清理: [true/false]（默认 true，测试完成后自动清理）
   # ============================================================
   ```

3. **存储路径**：根据文件类型存放到对应子目录
   - `temp/test_scripts/` - 临时测试脚本
   - `temp/test_data/` - 临时测试数据
   - `temp/test_output/` - 临时测试输出
   - `temp/debug/` - 调试临时文件

4. **记录到清单**：更新《临时文件清单》

### 2. 测试完成自动清理

#### 2.1 自动清理触发条件
- **默认行为**：用户未特殊要求时，测试完成后自动删除临时测试程序
- **触发时机**：
  1. 测试脚本执行完成且退出码为 0（表示测试成功）
  2. 用户确认测试效果符合预期
  3. 临时文件无特殊保留需求

#### 2.2 自动清理流程
1. **检测测试完成**：监控测试脚本执行状态
2. **验证测试结果**：检查测试是否成功完成
3. **确认用户需求**：检查是否有特殊保留要求
4. **执行清理**：删除临时测试文件
5. **更新记录**：在清理日志中记录清理信息

#### 2.3 清理确认机制
- **静默清理**：默认情况下，测试成功后自动清理，无需用户确认
- **用户确认**：如果测试文件较大（> 10MB）或包含重要数据，会提示用户确认后再清理
- **取消清理**：用户可以通过添加 `# 自动清理: false` 到文件头来禁用自动清理

### 2. 保留期限管理

#### 2.1 按「测试阶段 + 文件价值」的分层保留期限

| 临时文件类型 | 测试阶段 | 基础保留期限 | 最长期限 |
|------------|---------|------------|---------|
| 快速验证脚本（单次算法验证） | 单元测试 | 1 天 | 3 天 |
| 快速验证脚本（功能验证） | 集成测试 | 3 天 | 7 天 |
| 测试数据文件（核心场景） | 系统测试 / 验收测试 | 14 天 | 21 天 |
| 测试数据文件（普通场景） | 单元 / 集成测试 | 7 天 | 14 天 |
| 调试输出文件（线上问题排查） | 生产调试 | 7 天 | 14 天 |
| 调试输出文件（普通调试） | 开发调试 | 3 天 | 7 天 |
| 临时备份文件（核心版本） | 版本发布前 | 14 天 | 21 天 |
| 临时备份文件（普通版本） | 日常开发 | 7 天 | 14 天 |
| 其他临时文件（关联里程碑） | 里程碑测试 | 15 天 | 30 天 |
| 其他临时文件（一般场景） | 常规测试 | 7 天 | 30 天 |

#### 3.2 动态延期规则

| 文件价值等级 | 延期次数 | 单次最长延期 | 累计最长延期 |
|------------|---------|------------|------------|
| 低价值（单次验证） | 0 次 | - | - |
| 中价值（常规测试） | 1 次 | 7 天 | 7 天 |
| 高价值（核心测试 / 问题排查） | 2 次 | 10 天 | 20 天 |

### 4. 清理机制

#### 4.1 定期清理
- **每日清理**：清理已过期的临时文件
- **每周清理**：提醒即将到期的临时文件
- **月度清理**：全面检查所有临时文件

#### 4.2 自动清理机制
- **文件到期**：低价值文件自动删除，中/高价值文件发送清理提醒
- **文件到期超过 3 天**：移动到 `archive/` 目录
- **文件到期超过 7 天**：从 `archive/` 目录删除
- **磁盘使用率 > 85%**：优先清理低价值、超期未处理的临时文件
- **磁盘使用率 > 90%**：清理所有超期文件，包括部分中价值文件

### 5. 归档机制

#### 5.1 归档类型
- **短期归档**：30 天，适用于常规测试复盘
- **长期归档**：90 天，适用于核心问题排查、客户验收相关
- **特殊保留**：最长 180 天，需双审批

#### 5.2 归档流程
1. 填写归档申请表
2. 获得项目负责人审批
3. 移动文件到 `temp/archive/` 目录
4. 更新《临时文件清单》
5. 设置归档期限

## 配置文件管理
- **配置文件位置**：`.trae/skills/temp-file-manager/config.json`
- **存储内容**：
  - 临时文件清单
  - 清理日志
  - 归档记录
  - 配置参数

## 错误处理
- **文件命名违规** → 自动修正命名格式
- **存储路径错误** → 自动移动到正确目录
- **超期未清理** → 发送清理提醒
- **磁盘空间不足** → 优先清理低价值文件

## 核心原则
- **最小必要原则**：保留期限 =「完成测试 / 复盘所需的最短时间 + 3 天缓冲」
- **分级管控原则**：按文件价值、测试阶段差异化设定期限
- **自动化 + 人工结合**：基础清理自动化，高价值文件人工评估
- **动态优化原则**：基于实际使用数据持续调整
